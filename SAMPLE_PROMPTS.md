# 📋 Sample Prompts Used Internally

This file documents all system prompts and user message templates used by each agent.
Useful for debugging, tuning, and understanding agent behavior.

---

## Agent 1 — Job Analyzer

### System Prompt Summary
The agent is instructed to think simultaneously as:
- An ATS system (keyword extraction)
- A recruiter (surface-level requirements)
- A hiring manager (implicit expectations)

### Key Prompt Engineering Techniques Used
1. **Persona stacking** — "think as X, Y, and Z simultaneously"
2. **Explicit schema enforcement** — exact JSON schema in prompt prevents hallucination
3. **Hidden expectations list** — teaches model to look for implied requirements
4. **Instruction examples** — shows what "fast-paced startup" implies

### Sample User Message
```
Analyze this job description and return structured JSON output.
Be thorough — include hidden expectations that aren't explicitly stated but are clearly implied.

JOB DESCRIPTION:
[job text here]

Return ONLY valid JSON matching the schema. No other text.
```

## Agent 2 — Profile Matcher

### System Prompt Summary
Acts as a senior career strategist with recruiter background.
Scores across 4 dimensions with explicit rubric.

### Scoring Rubric (in prompt)
- `skill_match_score`: % of required + preferred skills candidate has
- `experience_match_score`: quality and relevance of experience
- `education_match_score`: education alignment (lenient — experience > education)
- `overall_score`: weighted average (skills 40%, experience 50%, education 10%)

### Key Techniques
1. **Explicit weighted rubric** — prevents arbitrary scoring
2. **"Be honest but strategic"** — stops over-deflation of scores
3. **relevant_experience instruction** — "copy exact bullet text"
4. **Calibration anchor** — "65% match is worth applying to; 30% is not"

### Sample User Message
```
Score this candidate against the job requirements.

CANDIDATE PROFILE:
[formatted candidate text]

JOB REQUIREMENTS:
[JSON job requirements]

RELEVANT PAST APPLICATION CONTEXT:
[memory retrieval results if available]
```

## Agent 3 — Resume Generator

### System Prompt Summary
Expert resume writer. Enforces ATS optimization AND human readability.

### Key Prompt Engineering Techniques
1. **Numbered principles** — 7 explicit rules the model must follow
2. **Anti-pattern prohibition** — "NO generic filler phrases" with examples
3. **ATS rules section** — separate explicit ATS instructions
4. **Tone injection** — "sound like a human, not generated by AI"
5. **Avoidance word list** — "avoid: passionate about, dynamic, results-driven..."
6. **Format specification** — exact Markdown structure with section headers

### Sample User Message
```
Write a tailored, ATS-optimized resume for this candidate applying to this specific role.

TARGET ROLE: [title] at [company]

CANDIDATE INFO:
[full candidate info]

JOB REQUIREMENTS:
- Required skills: [...]
- ATS keywords to include: [...]

MOST RELEVANT EXPERIENCE BULLETS (prioritize these):
[bullets from ProfileMatcher.relevant_experience]

CANDIDATE STRENGTHS FOR THIS ROLE:
[ProfileMatch.strengths_summary]
```

## Agent 4 — Cover Letter Generator

### System Prompt Summary
"Master cover letter writer" who prioritizes story over resume summary.

### The Cover Letter Formula (in prompt)
1. **OPENING** — specific company hook, never "I am writing to apply"
2. **WHY YOU** — one specific achievement story
3. **WHY THEM** — genuine research signal
4. **BRIDGE** — connect skills to their needs
5. **CLOSE** — confident, not desperate

### Style Rules (in prompt)
- 250-350 words exactly
- No clichés (list of 6 banned phrases)
- Don't start any paragraph with "I"
- Vary sentence structure

### Key Techniques
1. **Negative examples** — "Never start with 'I am writing to apply...'. Never."
2. **Formula structure** — reduces generic output by giving concrete blueprint
3. **Tone determination** — startup vs enterprise vs default (dynamic per company)
4. **Company-specific injection** — forces research-signal language

## Agent 5 — Application Agent

### System Prompt Summary
Writes professional application emails. 150-200 words max.

### Email Rules (in prompt)
- Good subject: "Application: Senior Backend Engineer — Jane Smith"
- Bad subject: "Job Application"
- Opening: who you are + what you're applying for
- Middle: ONE specific qualification
- Close: forward-looking, professional

### Key Techniques
1. **Good/bad examples** — concrete contrast for subject line quality
2. **Hard word limit** — "150-200 words maximum. Hiring managers are busy."
3. **JSON output** — subject + body + attachments_note as separate fields

## Critic Agent (Bonus)

### Resume Critic Prompt Summary
Senior recruiter reviewing resumes. Brutal honesty. Specific, not vague.

### Key Instruction
"'Bullet point 3 in Experience 1 lacks a quantified outcome' is useful.
'The resume is good' is not useful."

### Cover Letter Critic Prompt Summary
Hiring manager who has read 10,000 cover letters.

### Red Flags Checklist (in prompt)
- Generic opening detection
- Resume restating detection
- Missing company-specific research
- Cliché language scan
- Wrong length check (< 200 or > 400 words)

## Memory Context Injection Pattern

When similar past applications are found in FAISS, this context is injected
into the Profile Matcher's user message:

```
RELEVANT PAST APPLICATION CONTEXT:
Similar past applications (for context):
  • Senior Engineer @ BigCo | Match score: 72% | Outcome: interview
    Notes: Emphasized distributed systems; de-emphasized frontend experience
  • Backend Lead @ StartupXY | Match score: 85% | Outcome: offer
    Notes: Highlighted Kafka + Python combination; company culture match
```

This allows the matcher to calibrate its scoring and strategy based on
what has worked/failed in similar contexts.

## Prompt Improvement Suggestions

1. **Few-shot examples** — Add 2-3 example resume bullets before/after transformation
   to dramatically improve output quality on the resume generator.

2. **Chain-of-thought for scoring** — Ask the Profile Matcher to reason step-by-step
   before outputting scores (`"reasoning": "..."` field before `"overall_score"`).

3. **Structured critique format** — In the Critic agent, ask for before/after pairs:
   `{"original": "...", "improved": "..."}` for each suggestion.

4. **Company research tool** — Add a web search tool call before the Cover Letter agent
   to retrieve real company news/blog posts for genuine research signals.

5. **ATS simulation pass** — Add a dedicated ATS simulation agent that scores
   the generated resume against common ATS patterns and keyword density rules.

6. **Tone calibration examples** — Add 2-3 example paragraphs per tone type
   (startup/enterprise/default) as few-shot examples in the Cover Letter prompt.
